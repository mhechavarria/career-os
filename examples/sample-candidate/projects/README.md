# Projects

CV/portfolio-facing engineering projects. Each entry is one markdown file with frontmatter
(`name`, `slug`, `status`, `role`, `stack`, `links`, `visibility`, `period`), an
outcome-first summary, and deeper context/architecture for interview prep.

## Index

- **tide-gauge** — shipped · public · coastal water-level IoT logger (ESP32/C · LoRaWAN · Go · Grafana) — [tide-gauge.md](tide-gauge.md)
- **kvlite** — shipped · public · embeddable LSM-tree key-value store in Go — [kvlite.md](kvlite.md)
- **promql-lab** — shipped · public · observability teaching sandbox (Prometheus · Grafana) — [promql-lab.md](promql-lab.md)

## Conventions

- **One file per project**, slug = kebab-case, filename = `<slug>.md`.
- **`visibility`**: `public` · `private` · `private-until-public`.
- **Outcomes first.** Lead each entry with tight, quantified, outcome-framed bullets; keep
  longer context and architecture below for interview depth.
