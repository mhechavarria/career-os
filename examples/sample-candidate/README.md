# Sample Candidate — Worked Example & End-to-End Test Fixture

This directory is a **complete, fully worked Career OS instance for a fictional
engineer, "Jordan Rivera."** It exists for two reasons:

1. **A worked example.** New users can see what a filled-out instance looks like —
   profile, five roles of experience, impact libraries, projects, a master CV, and
   tailored CV variants — instead of starting from the blank top-level placeholders.
2. **An end-to-end test fixture.** [`../run_pipeline.py`](../run_pipeline.py) drives
   every script in the pipeline against this data and asserts that each stage works:
   CV render → keyword gap → application bootstrap → pipeline report.

> **Everything here is invented.** Jordan Rivera, the companies (Heliograph, Nimbus
> Freight, Coriolis Analytics, Tessera Health, Vela Robotics), the projects, the
> metrics, and the contact details are all fictional. Any resemblance to real people
> or companies is coincidental. This is sample data, not a real CV.

## Layout

This mirrors the standard Career OS layout, one level down:

```
sample-candidate/
  profile/       about, headline, evolution, principles, skills
  experience/    one file per role (5 roles, 2013–present)
  impacts/       per-company impact libraries + aggregate index + brag-doc
  projects/      three open-source project writeups + index
  cv/            master.md + versions/ (three tailored variants)
  jds/           three target job descriptions (plain text)
  applications/  pipeline.md dashboard (application files are generated by the runner)
```

## Run the end-to-end test

From the repository root:

```bash
python3 examples/run_pipeline.py          # run every stage, print PASS/FAIL
python3 examples/run_pipeline.py --keep    # keep the temp work dir for inspection
```

The runner assembles a throwaway working copy (this example's data + fresh copies of
`scripts/`), then runs the real scripts against it. It never modifies this directory
or the repository's top-level `cv/`, `applications/`, or `jds/` folders.
