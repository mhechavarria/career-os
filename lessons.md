# Lessons — the Judgment Flywheel

> The compounding memory of your job search. Company folders and CVs are rebuilt each run;
> this file isn't. It's how run N+1 inherits what runs 1…N taught you — a flywheel, not a
> flat log.
> **The loop, in three moves:**
> **Capture** — when an application reaches a terminal outcome (rejection, ghost,
> withdrawal, or a win — offer/placed), the retro in its application file records what
> happened and what predicted it (Phase 9).
> **Distill** — occasionally, when the same lesson recurs across retros, promote it here as
> a durable, generalized principle. Not every retro graduates; only the ones that repeat.
> **Apply** — the triage gate (Phase 7.0), research (Phase 7), and interview prep (Phase 8)
> read this file first, so past lessons sharpen the next run before you spend effort.
> Keep it short and principled — each lesson is a line or two: the pattern, then what to do
> differently. If this file turns into a diary, it stops getting read.
